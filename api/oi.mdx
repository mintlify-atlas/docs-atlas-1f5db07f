## Overview

The `OI` class (Operator Interface) manages all gamepad input for the robot. It provides a clean interface for reading controller buttons and analog sticks with automatic deadzone application.

**Package:** `frc.robot.gamepad`

## Constructor

### OI()

Creates a new Operator Interface instance and initializes the drive gamepad on the USB port specified in `GamepadConstants.USB_PORT`.

```java
public OI()
```

## Analog Stick Methods

All analog methods automatically apply deadzone filtering using `Constants.DEADZONE`.

### getLeftDriveX()

Returns the X-axis value of the left analog stick (horizontal movement).

```java
public double getLeftDriveX()
```

<ParamField path="return" type="double">
  Left stick X-axis value from -1.0 to 1.0 (left to right) with deadzone applied
</ParamField>

### getLeftDriveY()

Returns the Y-axis value of the left analog stick (vertical movement), inverted.


<ParamField path="return" type="double">
  Left stick Y-axis value from -1.0 to 1.0 (up to down) with deadzone applied and inverted
</ParamField>

**Note:** This method applies an additional negation to invert the axis direction.

## Deadzone Logic

### applyDeadzone(double value, double deadzone)

Private helper method that filters small analog stick movements to prevent drift.

```java
private double applyDeadzone(double value, double deadzone)
```

<ParamField path="value" type="double">
  Raw analog input value from -1.0 to 1.0
</ParamField>

<ParamField path="deadzone" type="double">
  Threshold below which input is zeroed (typically `Constants.DEADZONE = 0.06`)
</ParamField>

<ParamField path="return" type="double">
  Input value if `|value| >= deadzone`, otherwise `0.0`
</ParamField>

**Logic:** Returns 0.0 if the absolute value is less than the deadzone threshold, otherwise returns the original value unchanged.
