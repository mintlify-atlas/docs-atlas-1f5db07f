## Prerequisites

Before you begin, ensure you have the following installed:

- **WPILib 2020** installed (see [detailed setup guide](/setup))
- **Java 11** or newer
- **Git** for version control
- **RoboRIO** configured and connected to your network

## Quick setup

<Steps>
  <Step title="Clone the repository">
    Clone the Agrobotics repository to your local machine:
    
    ```bash
    git clone https://github.com/ozzyDev27/Agrobotics.git
    cd Agrobotics
    ```
  </Step>

  <Step title="Configure team number">
    Edit `.wpilib/wpilib_preferences.json` and set your team number:
    
    ```json
    {
      "enableCppIntellisense": false,
      "currentLanguage": "java",
      "projectYear": "2020",
      "teamNumber": 1234
    }
    ```
    
    Replace `1234` with your FRC team number.
  </Step>

  <Step title="Build the project">
    Build the robot code using Gradle:
    
    ```bash
    ./gradlew build
    ```
    
    This compiles the Java code and packages it into a deployable JAR file.
  </Step>

  <Step title="Deploy to RoboRIO">
    Deploy the code to your RoboRIO:
    
    ```bash
    ./gradlew deploy
    ```
    
    <Note>Make sure your RoboRIO is connected to your network and powered on before deploying.</Note>
  </Step>
</Steps>

## Verify deployment

After deployment completes successfully, verify that:

- The robot code is running on the RoboRIO
- Driver Station shows the robot as connected
- No errors appear in the console
- The NavX gyroscope initializes successfully

## Common commands

Here are some useful Gradle commands for robot development:

```bash
# Build the project
./gradlew build

# Deploy to RoboRIO
./gradlew deploy

# Deploy with debug enabled
./gradlew deploy -PdebugMode

# Clean build artifacts
./gradlew clean

# Run in simulation mode
./gradlew simulateJava
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="RoboRIO not found">
    Ensure your RoboRIO is:
    - Powered on
    - Connected to the same network as your computer
    - Imaged with the correct firmware
    - Configured with your team number
  </Accordion>

  <Accordion title="Build fails">
    Try cleaning the project first:
    ```bash
    ./gradlew clean build
    ```
    
    If the issue persists, check that you have Java 11 installed and WPILib 2020.3.2.
  </Accordion>

  <Accordion title="Deploy timeout">
    Check your network connection and firewall settings. The deploy process requires network access to the RoboRIO at `roboRIO-TEAM-frc.local` or `10.TE.AM.2`.
  </Accordion>

  <Accordion title="NavX initialization fails">
    Verify that:
    - The NavX is properly connected to the SPI port (MXP)
    - The NavX firmware is up to date
    - No other devices are using the same SPI port
  </Accordion>
</AccordionGroup>
